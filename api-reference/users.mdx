---
title: "Users"
description: "List users within an organization, with optional filtering by user ID or email."
---

Users represent individual accounts in Airbyte. The Users API allows organization admins to list and filter users within their organization.

<Info>
  Only organization admins can list users. You can only list users in your own organization unless you have elevated permissions.
</Info>

## Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/users` | List users within an organization |

## List users

**`GET /users`**

List all users within the specified organization. Supports filtering by user IDs or email addresses.

<ParamField query="organizationId" type="string" required>
  UUID of the organization to list users for.
</ParamField>

<ParamField query="ids" type="string[]">
  Filter by specific user UUIDs.
</ParamField>

<ParamField query="emails" type="string[]">
  Filter by specific email addresses.
</ParamField>

```bash
curl --request GET \
  --url 'https://api.airbyte.com/v1/users?organizationId=9924bcd0-99be-453d-ba47-c2c9766f7da5' \
  --header 'Authorization: Bearer <token>'
```

### Response

<ResponseField name="data" type="object[]" required>
  Array of user objects.
  <Expandable title="user object">
    <ResponseField name="id" type="string" required>
      Internal Airbyte user ID (UUID).
    </ResponseField>
    <ResponseField name="name" type="string" required>
      Display name of the user.
    </ResponseField>
    <ResponseField name="email" type="string" required>
      Email address of the user.
    </ResponseField>
  </Expandable>
</ResponseField>

```json
{
  "data": [
    {
      "id": "7d08fd6c-531e-4a00-937e-3d355f253e63",
      "name": "Jane Smith",
      "email": "jane@acme.com"
    }
  ]
}
```

## User status values

User accounts can be in one of the following states:

| Status | Description |
|---|---|
| `invited` | User has been invited but has not yet accepted |
| `registered` | User has registered and is active |
| `disabled` | User account has been disabled |

## Auth providers

Users may authenticate through the following providers:

- `airbyte` (default)
- `google_identity_platform`
- `keycloak`
