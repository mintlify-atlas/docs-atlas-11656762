---
title: "Destinations"
description: "List, create, update, and delete data destinations in Airbyte."
---

A destination is a configured connector that writes data to an external system. Each destination belongs to a workspace and has a connector-specific `configuration` object.

## Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/destinations` | List destinations |
| `POST` | `/destinations` | Create a destination |
| `GET` | `/destinations/{destinationId}` | Get destination details |
| `PATCH` | `/destinations/{destinationId}` | Partially update a destination |
| `PUT` | `/destinations/{destinationId}` | Fully replace a destination |
| `DELETE` | `/destinations/{destinationId}` | Delete a destination |

## List destinations

**`GET /destinations`**

<ParamField query="workspaceIds" type="string[]">
  Filter by workspace UUIDs. Empty list returns all allowed workspaces.
</ParamField>

<ParamField query="includeDeleted" type="boolean" default="false">
  Include soft-deleted destinations.
</ParamField>

<ParamField query="limit" type="integer" default="20">
  Maximum results to return. Range: 1–100.
</ParamField>

<ParamField query="offset" type="integer" default="0">
  Offset for pagination.
</ParamField>

```bash
curl --request GET \
  --url 'https://api.airbyte.com/v1/destinations?workspaceIds=744cc0ed-7f05-4949-9e60-2a814f90c035' \
  --header 'Authorization: Bearer <token>'
```

## Create a destination

**`POST /destinations`**

Creates a destination given a name, workspace ID, and a configuration blob matching the connector specification.

<ParamField body="name" type="string" required>
  Name of the destination, e.g. `dev-postgres`.
</ParamField>

<ParamField body="workspaceId" type="string" required>
  UUID of the workspace to create the destination in.
</ParamField>

<ParamField body="configuration" type="object" required>
  Connector-specific configuration. Must include `airbyte_destination_name` identifying the connector type.
</ParamField>

<ParamField body="definitionId" type="string">
  UUID of the connector definition. One of `configuration.destinationType` or `definitionId` must be provided.
</ParamField>

<ParamField body="resourceAllocation" type="object">
  Resource requirements for the destination's sync workers.
  <Expandable title="properties">
    <ParamField body="default" type="object">
      Default resource requirements applied to all job types.
    </ParamField>
    <ParamField body="jobSpecific" type="object[]">
      Per-job-type resource overrides.
    </ParamField>
  </Expandable>
</ParamField>

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.airbyte.com/v1/destinations \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{
    "name": "Postgres",
    "workspaceId": "2155ae5a-de39-4808-af6a-16fe7b8b4ed2",
    "configuration": {
      "airbyte_destination_name": "postgres",
      "host": "localhost",
      "port": 5432,
      "database": "postgres",
      "username": "postgres",
      "schema": "public"
    }
  }'
```

</CodeGroup>

### Response

<ResponseField name="destinationId" type="string" required>
  UUID of the created destination.
</ResponseField>

<ResponseField name="name" type="string" required>
  Name of the destination.
</ResponseField>

<ResponseField name="destinationType" type="string" required>
  Connector type identifier, e.g. `postgres`.
</ResponseField>

<ResponseField name="definitionId" type="string" required>
  UUID of the connector definition.
</ResponseField>

<ResponseField name="workspaceId" type="string" required>
  UUID of the containing workspace.
</ResponseField>

<ResponseField name="configuration" type="object" required>
  Connector-specific configuration values. Secret fields are masked as `***`.
</ResponseField>

<ResponseField name="createdAt" type="integer" required>
  Unix timestamp (ms) when the destination was created.
</ResponseField>

<ResponseField name="resourceAllocation" type="object">
  Resource requirements for sync workers.
</ResponseField>

## Get a destination

**`GET /destinations/{destinationId}`**

<ParamField path="destinationId" type="string" required>
  UUID of the destination.
</ParamField>

<ParamField query="includeSecretCoordinates" type="boolean">
  If `true`, return secret coordinate references instead of `***` for sensitive fields.
</ParamField>

```bash
curl --request GET \
  --url https://api.airbyte.com/v1/destinations/18dccc91-0ab1-4f72-9ed7-0b8fc27c5826 \
  --header 'Authorization: Bearer <token>'
```

## Update a destination (partial)

**`PATCH /destinations/{destinationId}`**

Updates only the fields provided in the request body.

<ParamField path="destinationId" type="string" required>
  UUID of the destination.
</ParamField>

<ParamField body="name" type="string">
  New name for the destination.
</ParamField>

<ParamField body="configuration" type="object">
  Updated connector configuration.
</ParamField>

```bash
curl --request PATCH \
  --url https://api.airbyte.com/v1/destinations/18dccc91-0ab1-4f72-9ed7-0b8fc27c5826 \
  --header 'Authorization: Bearer <token>' \
  --header 'Content-Type: application/json' \
  --data '{"name": "Updated Destination Name"}'
```

## Replace a destination

**`PUT /destinations/{destinationId}`**

Fully replaces the destination configuration. All fields are overwritten.

<ParamField path="destinationId" type="string" required>
  UUID of the destination.
</ParamField>

<ParamField body="name" type="string" required>
  Name for the destination.
</ParamField>

<ParamField body="configuration" type="object" required>
  Complete connector configuration.
</ParamField>

## Delete a destination

**`DELETE /destinations/{destinationId}`**

<ParamField path="destinationId" type="string" required>
  UUID of the destination to delete.
</ParamField>

Returns `204 No Content` on success.

```bash
curl --request DELETE \
  --url https://api.airbyte.com/v1/destinations/18dccc91-0ab1-4f72-9ed7-0b8fc27c5826 \
  --header 'Authorization: Bearer <token>'
```
